**ai-blog-assistant**
从0到1完整项目工作流 · AI产品经理视角 · Vibe Coding驱动

**产品定位**：帮助博主、营销人员等内容创作者用AI克服写作灵感枯竭，从0快速生成兼具个人风格的高质量博客文章，节省80%写作时间。

**变现模式**：Freemium（免费试用 + 订阅制Pro版）

**核心差异化**：风格注入（Few-shot）+ Token级配额防滥用 + 内容合规护栏 + 语义相似度衡量AI价值

### 3.4 核心Prompt资产（PM必备）

#### 📌 标题生成

```
你是一位专业的内容策略师，擅长为中文博客创作吸引人的标题。
用户想写关于「{topic}」的博客文章，目标读者是{audience}。

请生成10个吸引人的标题，要求：
1. 每个标题都有明确的钩子（数字、问题、对比、悬念之一）
2. 包含核心关键词「{topic}」
3. 长度控制在15-25字
4. 不同标题覆盖不同角度（实用干货、深度思考、对比分析等）
5. 避免标题党，标题要能兑现文章承诺

输出格式：标题 + 一句话说明（为什么有吸引力）
```

#### 📌 大纲生成

```
基于以下标题生成专业的博客大纲：
标题：{title}
文章长度：{length}（短=1000字/中=2000字/长=3000字）

输出结构化JSON：
{
  "intro": "引言主旨",
  "sections": [
    {
      "title": "章节标题",
      "points": ["子要点1", "子要点2", "子要点3"]
    }
  ],
  "conclusion": "结论主旨"
}
```

#### 📌 风格注入扩写（Pro）

```
你是一位专业博客作者。以下是该作者的写作风格特征：
{style_card}

参考风格示例（节选）：
{style_sample_excerpt}
（取参考文中最典型的1-2段，约300字）

请按照上述风格，扩写以下大纲章节为完整段落（{word_count}字）：
章节：{section_title}
要点：{section_points}

要求：保留作者风格，不生硬模仿，内容真实有用。
```

#### 📌 段落扩写（标准，Multi-Agent 三步协作）

**Step 1 — Research Agent（研究员）**
```
你是一位专业研究员，擅长为博客文章提供数据支撑。

当前章节：{section_title}
章节要点：{section_points}

请提供3-5条与本章节高度相关的客观事实、数据或案例，要求：
1. 每条数据需注明来源类型（行业报告/学术研究/权威媒体）
2. 优先选择近3年内的数据
3. 避免过于宽泛的常识性内容
4. 数据要能直接支撑章节论点

输出格式：编号列表，每条50字以内
```

**Step 2 — Writer Agent（撰稿人）**
```
你是一位专业博客撰稿人，负责将大纲扩写为完整段落。

章节标题：{section_title}
章节要点：{section_points}
数据支撑（Research Agent提供）：
{research_data}

请扩写为300-500字的完整章节内容，要求：
1. 开篇有吸引力，结尾有总结
2. 自然融入数据支撑，不堆砌
3. 段落之间逻辑流畅，有过渡
4. 语气专业但不生硬，适合博客阅读
```

**Step 3 — Reviewer Agent（主编，LLM-as-Judge）**
```
你是一位资深博客主编，负责审核文章质量。

原始大纲要点：{section_points}
撰稿人初稿：{draft_content}

请从以下维度评审初稿（每项1-5分，5分最高）：
1. 大纲覆盖度：是否覆盖了全部要点？
2. 逻辑流畅度：段落之间是否自然衔接？
3. 数据融合度：数据引用是否自然且有说服力？
4. 语气一致性：整体语气是否统一？

若所有维度均≥4分：输出 {"approved": true, "content": "原稿内容"}
若任一维度<4分：输出 {"approved": false, "feedback": "具体修改意见", "focus": "需重点改进的方面"}
```

#### 📌 全文润色

```
你是一位专业文字编辑。请对以下博客文章进行{tone}风格的润色优化：

原文：
{draft}

润色要求：
1. 保持作者核心观点不变
2. 提升句子多样性（长短句交替）
3. 优化段落衔接，使行文更流畅
4. 修正语病，统一语气
5. 润色强度：{intensity}（轻度=微调措辞/中度=优化句式/深度=重组段落）

输出完整润色后的文章，不添加额外说明。
```

## PHASE 4 · 测试与迭代（持续进行）

### 4.1 关键指标体系（OKRs）

| 类型 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| 北极星 | 文章完成率 | >40% | 用户完成全流程写作的比例 |
| 增长 | 注册用户 | 3月500+ | 上线后3个月累计注册数 |
| AI质量 | AI满意度评分 | ≥3.8/5 | 用户对AI生成内容的评分 |
| AI效用★ | 语义相似度 KR1.4 | ≥0.75 | 用户发布版 vs AI初稿余弦相似度（0.75-0.90为健康区间） |
| 转化 | Free→Pro转化 | >5% | 免费用户升级付费比例 |
| 成本健康 | API成本/MRR | ≤30% | 月度GLM API费用占月收入比 |
| 技术 | API响应时间 | <3s | P95响应时间 |

> ★ 语义相似度说明：用户导出/保存文章时，对比AI初稿与最终版的向量余弦相似度。
> <0.50=AI基本无效（立即排查Prompt）；0.75-0.90=健康区间；>0.90=用户几乎未改（结合满意度区分好懒）。
> 使用 text-embedding-3-small 计算，成本约$0.02/1M tokens。
