## 1. 摘要

AI写作助手（ai-blog-assistant）是一款面向博主、营销人员等内容创作者的AI写作工具。它通过结构化的四步写作流程（选题 → 大纲 → 风格注入 → 扩写），帮助用户从零快速生成兼具个人风格的高质量博客文章，核心解决"从0写作难、灵感枯竭、AI味太重"三个高频痛点。产品采用 Freemium 商业模式（免费试用 + 订阅制Pro版），配额以 Token 字重计费（非篇数），内置内容合规护栏。目标在上线后3个月内获得 500+ 注册用户、5% 免费转付费转化率。

## 3. 背景

### 3.1 为什么做这件事？

内容创作已成为个人品牌建设和企业营销的核心手段。但对于大多数博主和营销人员来说，写作是一件痛苦的事情——不知道从哪里开始、面对空白页面不知所措、花大量时间在结构组织上而非真正的价值输出上。

- **行业数据**：82% 的内容创作者表示"写作启动难"是他们最大的障碍（来源：内容营销协会）
- **市场规模**：全球AI写作工具市场预计2026年达 $18亿，年增长率超40%
- **中文市场空白**：现有工具（Jasper、Copy.ai）以英文为主，中文体验差，针对博客全流程的垂类工具几乎空白
- **新增痛点**：已有AI写作工具的用户反映"AI味太重"是第一弃用原因——内容像模板、失去个人声音

### 3.2 为什么是现在？

1. **Claude API性能突破**：Claude 3.5 Sonnet的中文写作质量已达到可商业化水准
2. **技术成本下降**：API调用成本降低80%，Freemium模型可行
3. **用户教育完成**：ChatGPT的普及已让目标用户接受并期待AI辅助写作
4. **Few-shot风格迁移可行**：通过3-5篇参考文章即可在Prompt层实现风格模仿，无需Fine-tune，V1即可落地

## 4. 目标

### 4.1 产品目标

**核心目标**：打造一款让博主/营销人员用AI克服"写作启动难"的垂类工具，且生成内容保留用户个人风格，成为中文博客创作领域的首选AI助手。

**对用户的价值**：
- 从"不知道写什么"到"完成一篇文章"的时间从3小时缩短到30分钟
- 生成内容保留个人风格，用户无需大量修改即可发布
- 降低写作心理门槛，提升创作频率

**对公司的价值**：
- 验证 AI + 内容创作 的商业模式
- 积累写作领域的用户数据和Prompt资产
- 建立可扩展的 SaaS 产品基础

## 7. 解决方案

### 7.1 用户体验流程（UX Flow）

#### 核心创作流程（Happy Path）

```
Step 1: 主题输入
  用户输入关键词或描述（如"AI工具推荐"）
  → AI生成10个标题建议
  → 用户选择1个或自定义

Step 2: 大纲确认
  AI生成结构化大纲（引言 + 3-5章节 + 结论）
  → 用户可拖拽调整顺序
  → 用户可增删章节
  → 用户确认大纲

Step 2.5: 风格注入（新增，Pro专属）
  用户可选择：
  a) 跳过（使用默认专业风格）
  b) 从历史文章中选取1-3篇作为风格参考
  c) 粘贴文本（不超过5000字）作为风格参考
  → 系统提取写作风格特征（句子长度、语气、用词习惯）
  → 生成风格描述摘要供用户确认

Step 3: 全文生成
  AI逐段扩写（每段300-500字）
  → 携带风格参考进行Few-shot生成
  → 进度条实时显示
  → 全文展示在可编辑区
  → 一键全文润色/SEO优化

Step 4: 导出
  支持 Markdown / 富文本复制 / Word下载
  → 后台记录"用户修改量"（用于计算语义相似度）
```

#### 关键页面设计要点

**Landing Page**
- 首屏：一句话价值主张 + 动态演示GIF + 免费试用CTA
- 展示对比：写作前（空白痛苦） vs 写作后（完成满足）
- 社会证明：早期用户评价 + 已生成文章数量

**工作台（编辑器主界面）**
- 三栏布局：左侧步骤导航 | 中间AI输出区 | 右侧编辑区
- 每步骤有清晰的"下一步"引导
- 显示剩余 Token 配额（Free用户强化升级意识）

**定价页**
- Free / Pro / Team 三栏对比
- 突出Pro的"最受欢迎"标签
- 年付优惠（相当于打7折）显著展示

### 7.2 核心功能详细说明

#### 功能 F01：博客标题生成

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户输入主题关键词，AI生成10个差异化标题 |
| 输入 | 主题关键词（必填）、文章风格（可选：实用/深度/故事） |
| 输出 | 10个候选标题，附带简短说明（为什么这个标题有吸引力） |
| 设计原则 | 标题要有钩子（数字/问题/对比），覆盖不同角度 |
| 限制 | Free用户每次生成5个，Pro用户10个 |
| 边界情况 | 关键词过于宽泛时提示用户细化（如"写作"→"博客写作技巧"） |

**核心Prompt模板**：
```
你是一位专业的内容策略师，擅长为中文博客创作吸引人的标题。
用户想写关于「{topic}」的博客文章，目标读者是{audience}。
请生成10个吸引人的标题，要求：
1. 每个标题都有明确的钩子（数字、问题、对比、悬念之一）
2. 包含核心关键词"{topic}"
3. 长度控制在15-25字
4. 不同标题覆盖不同角度（实用干货、深度思考、对比分析等）
5. 避免标题党，标题要能兑现文章承诺
输出格式：标题 + 一句话说明（为什么有吸引力）
```

#### 功能 F02：文章大纲生成

| 属性 | 说明 |
|------|------|
| 功能描述 | 基于选定标题生成可编辑的结构化大纲 |
| 输入 | 选定的标题、文章长度（短/中/长） |
| 输出 | 引言 + 3-5个主章节（每节含2-3子要点）+ 结论 |
| 交互 | 支持拖拽排序、增加/删除章节、编辑章节标题 |
| 数据格式 | JSON结构，前端渲染为可交互树状列表 |

**核心Prompt模板**：
```
基于以下标题生成专业的博客大纲：
标题：{title}
文章长度：{length}（短=1000字/中=2000字/长=3000字）

输出结构化JSON：
{
  "intro": "引言主旨",
  "sections": [
    {
      "title": "章节标题",
      "points": ["子要点1", "子要点2", "子要点3"]
    }
  ],
  "conclusion": "结论主旨"
}
```

#### 功能 F03（新增）：风格注入（Few-shot Style Transfer）

| 属性 | 说明 |
|------|------|
| 功能描述 | 用户提供1-3篇参考文章，AI提取风格特征并在扩写时应用 |
| 可用版本 | Pro 版专属（Free版使用标准风格） |
| 输入方式 | a）从历史文章中选取；b）粘贴文本（上限5000字/篇） |
| 风格特征维度 | 句子平均长度、段落节奏、语气（正式/轻松/犀利）、常用词汇偏好、举例习惯 |
| 系统处理 | 提取后生成"风格卡片"（约200字摘要）供用户确认，不将原文传入生成Prompt |
| 隐私保护 | 参考文章不持久化存储，仅用于当次生成的Prompt构建 |
| 降级策略 | 风格提取失败时（参考文本质量差），自动降级为标准风格并提示用户 |

**核心Prompt模板**：
```
你是一位专业博客作者。以下是该作者的写作风格特征：
{style_card}

参考风格示例（节选）：
{style_sample_excerpt}  # 取参考文中最典型的1-2段，约300字

请按照上述风格，扩写以下大纲章节为完整段落（{word_count}字）：
章节：{section_title}
要点：{section_points}

要求：保留作者风格，不生硬模仿，内容真实有用。
```

#### 功能 F04：段落扩写（底层 Multi-Agent 协作）

| 属性 | 说明 |
|------|------|
| 功能描述 | 将大纲节点逐步扩写为完整段落；前端体验极简（一键触发），后端由三个 Agent 协作完成 |
| 触发方式 | 用户确认大纲后点击"一键生成全文"，或单独扩写某章节 |
| 输出 | 每个章节300-500字，风格统一（若已注入风格则遵循）、有数据支撑、逻辑流畅 |
| 流式输出 | 使用 Server-Sent Events，逐字输出，提升感知速度 |
| 质量保障 | 每段生成后显示"重新生成此段"按钮；Reviewer Agent 不达标自动重写（最多2轮） |

**底层架构升级（Multi-Agent 协作）**

当用户点击"一键扩写"时，系统后端启动三智能体协作流：

```
用户点击"一键扩写"
        │
        ▼
┌───────────────────────────────────────────────┐
│           Multi-Agent 编排层（后端）            │
│                                               │
│  Step 1: Research Agent（研究员）              │
│  ├── 接收：当前章节标题 + 要点                  │
│  ├── 执行：调用知识库 / 联网搜索                 │
│  └── 输出：3-5 条客观事实或数据支撑             │
│                    │                          │
│                    ▼                          │
│  Step 2: Writer Agent（撰稿人）                │
│  ├── 接收：研究员数据 + 用户风格卡片             │
│  ├── 执行：按风格撰写章节初稿                    │
│  └── 输出：300-500字初稿                       │
│                    │                          │
│                    ▼                          │
│  Step 3: Reviewer Agent（主编）                │
│  ├── 接收：初稿 + 大纲要点 + 风格要求            │
│  ├── 执行：LLM-as-Judge 评估                   │
│  │   · 是否覆盖大纲要点？                       │
│  │   · 是否符合设定语气？                       │
│  │   · 数据引用是否准确？                       │
│  ├── 达标 → 流式输出给前端                      │
│  └── 不达标 → 打回 Writer Agent 重写（最多2轮） │
└───────────────────────────────────────────────┘
        │
        ▼（SSE 流式）
  前端逐字展示最终内容
```

| Agent | 角色 | 使用模型 | 关键输入 | 关键输出 |
|-------|------|---------|---------|---------|
| Research Agent | 研究员 | glm-4-flash（快速） | 章节标题 + 要点 | 3-5 条事实/数据 |
| Writer Agent | 撰稿人 | glm-4.5-air（主力） | 研究数据 + 风格卡片 | 章节初稿 |
| Reviewer Agent | 主编（LLM-as-Judge） | glm-4.5-air（主力） | 初稿 + 评审标准 | 通过 or 修改意见 |

#### 功能 F05：全文润色优化

| 属性 | 说明 |
|------|------|
| 功能描述 | 对完整草稿进行语言润色和可读性优化 |
| 润色维度 | 语言流畅度、句子长度多样性、段落衔接、语气统一 |
| 输出形式 | 直接替换原文 + 高亮显示主要修改处（可选） |
| 用户控制 | 可选择润色强度：轻度/中度/深度 |

#### 功能 F06：SEO关键词建议

| 属性 | 说明 |
|------|------|
| 功能描述 | 分析文章内容，提供SEO优化建议 |
| 输出 | 核心关键词列表 + 建议在文中出现次数 + 缺失关键词提示 |
| 扩展（V2）| 接入实际搜索量数据（如百度指数API） |

#### 功能 F07（新增）：内容合规护栏

| 属性 | 说明 |
|------|------|
| **必要性** | **V1.0 强制上线，不可裁剪** |
| 触发时机 | 用户输入主题时 + AI生成内容后（双重检查） |
| 检查内容 | 政治敏感词、色情内容、人身攻击、版权违规声明 |
| 实现方式 | 第一层：本地敏感词库（维护成本低）；第二层：Claude自身安全拒答机制 |
| 用户提示 | 输入端拦截：提示"当前主题可能涉及敏感内容，请调整描述"；输出端拦截：不展示，记录日志，提示重新生成 |
| 日志要求 | 所有拦截事件必须记录（user_id, input_text_hash, timestamp, rule_triggered），供定期审查 |
| 不做的事 | 不做人工审核（成本不可控），不向用户展示具体拦截规则（防绕过） |

#### 功能 F08：文章导出

| 属性 | 说明 |
|------|------|

| 元数据 | 导出时包含文章标题、生成时间、字数统计 |

## 附录 A：Token 配额设计（替代篇数计费）

### 为什么用Token而非篇数？

用"篇数"计费的问题：用户可以要求生成超长文章（如10000字），一篇实际消耗10篇的Token量，Free版被薅光。

### 配额方案

| 方案 | Token配额/月 | 对应文章数（估算） | 说明 |
|------|------------|----------------|------|
| **Free** | **30,000 tokens** | **约3-5篇中长文（2000字/篇）** | 含输入+输出合计 |
| **Pro** | 无限 | 无限 | 受每分钟速率限制保护 |
| **Team** | 无限（5席位） | 无限 | 受每分钟速率限制保护 |

### Token监控机制

```
每次API调用完成后：
1. 记录 input_tokens + output_tokens 到 usage_logs 表
2. 累计当月已用 tokens，更新 user_quota 表
3. 每次请求前检查：剩余 tokens < 1000 时展示"额度预警"
4. 剩余 tokens = 0 时，拒绝生成请求并引导升级
5. 每月1日自动重置 Free 用户额度
```

### 成本参考（Claude 3.5 Sonnet 价格）

| 操作 | 预估 tokens | 预估成本（美元） |
|------|-----------|--------------|
| 生成10个标题 | ~500 | ~$0.002 |
| 生成大纲 | ~800 | ~$0.003 |
| 扩写一篇2000字文章 | ~8,000 | ~$0.024 |
| 全文润色 | ~5,000 | ~$0.015 |
| **一篇完整文章全流程** | **~15,000** | **~$0.045（约¥0.33）** |

Free版30k tokens/月 → 约2篇完整流程文章，相当于给用户体验产品价值，不让Pro用户感到不公平。

## 附录 B：内容合规护栏规格

### 实现方案（轻量优先）

```
Level 1 — 本地关键词过滤（同步，零成本）
  - 维护敏感词库（政治/色情/暴力三类，约500词）
  - 用户输入主题时实时检测
  - 命中 → 前端拦截，不发起API调用

Level 2 — Claude安全机制（异步）
  - Claude本身会拒绝生成违规内容
  - 捕获Claude的拒绝响应（含"I cannot"等模式）
  - 记录日志，展示友好提示

不做 Level 3（人工审核）：成本不可控，MVP阶段不值得投入
```

### 日志规格

```sql
-- moderation_logs 表
CREATE TABLE moderation_logs (
  id UUID PRIMARY KEY,
  user_id TEXT NOT NULL,
  input_hash TEXT NOT NULL,  -- 原始输入的SHA256，不存明文
  trigger_level INT,          -- 1=本地词库 2=Claude拒绝
  rule_id TEXT,               -- 触发的规则ID
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 附录 C：语义相似度指标实现规格

### 数据采集

```
触发时机：用户点击"导出/复制/保存"时
采集内容：
  - ai_draft_hash：AI初稿的SHA256（用于关联，不存全文）
  - user_final_embedding：用户最终文本的向量（1536维）
  - ai_draft_embedding：AI初稿的向量（1536维）
计算方式：cosine_similarity(user_final_embedding, ai_draft_embedding)
存储：写入 article_analytics 表，与 article_id 关联
```

### 指标解读

| 相似度范围 | 含义 | 行动 |
|-----------|-----|------|
| > 0.90 | 用户几乎未改 | 结合满意度区分：高满意=质量好；低满意=用户放弃但懒得改 |
| 0.75 - 0.90 | 健康区间，AI有效辅助 | 维持现状 |
| 0.50 - 0.75 | 用户改动较大 | 分析改动集中的章节，针对性优化Prompt |
| < 0.50 | AI基本无效 | 严重问题信号，立即排查当天Prompt表现 |

## 附录 D：核心用户故事（User Stories）

```
作为一名独立博主（小红书运营者），
我希望能输入一个话题关键词就得到10个有吸引力的标题，
这样我就不用花1小时苦想标题、直接进入写作状态了。
验收标准：
  - 生成时间 < 5秒
  - 10个标题风格各异（数字/问题/对比至少各1个）
  - 可以一键选择并进入下一步

作为一名Pro用户，我有自己独特的写作风格，
我希望AI能模仿我的风格生成内容，
这样读者看不出是AI写的，我也不用花时间"去AI味"。
验收标准：
  - 上传参考文章后，生成内容在语气/用词上可感知的接近参考文章
  - 满意度打分 ≥ 4/5（风格匹配维度）
  - 风格提取 + 生成总时间 < 20秒

作为产品运营，
我需要确保平台不生成违规内容导致API封号，
这样产品可以稳定运营不被下架。
验收标准：
  - 敏感词输入前端实时拦截，不发起API调用
  - 所有拦截事件有完整日志可追溯
  - 每周人工抽查10条拦截日志，误拦截率 < 5%
```
